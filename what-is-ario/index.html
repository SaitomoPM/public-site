Are you a Gateway Operator?

The Arweave protocol doesn‚Äôt exist on its own.

The various nodes on the ar.io Network, known as Gateways, are the interface between users and the Arweave network. Each Gateway acts like a ‚ÄúPermaweb Service Provider‚Äù and supports multiple, value-added, low-trust services for users and applications that need to read and write data on Arweave. Gateway services are modular, extendable, and configurable to include:

Arweave.net was successfully transferred over to the custody of ar.io Network in early 2022. As this occurred, new opportunities emerged. The first one was the need for a new protocol for open-source gateways to the permaweb. This protocol has been developed and gateways using the new ar.io Network protocol are in private beta testing.

  1. seeding layer one transactions to the greater Arweave network
  2. caching and serving chain and transaction data
  3. resolving friendly names to transaction data with the Arweave Name System
  4. indexing and querying layer one and two transactions
  5. proxy for Arweave nodes serving the Arweave HTTP API

Gateway operators keep the front doors to the permaweb open by running reliable and performant systems for their own use cases, their community or even the whole Permaweb.

Are you a Developer?

Arweave is easier, cheaper and more fun to build on than most developers ever imagined.

Within the Arweave community there are many great development tools to help build your app such as arweave.js, warp contracts, and many other community tools and protocols.

One of the premier tool for building on Arweave is the Arweave File System (ArFS). ArFS enhances the Arweave protocol by adding the ability to create data hierarchy (folders), private encryption, file versioning, controlled sharing and more. It lets you harness the power of the permaweb for the projects you have in mind.
